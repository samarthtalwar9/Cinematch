# CineMatch AI 🎬

CineMatch AI is a modern, premium **Machine Learning Movie Recommendation Engine**. It features a stunning cinematic user interface built with raw HTML/CSS/JS and an ultra-fast static dataset generated by **Scikit-Learn (Python)** algorithms.

This project was built as a highly optimized, deployment-ready ML showcase perfect for engineering portfolios. By shifting the heavy computational matrix operations into an offline training phase, the model's outputs are served via a static `recommendations.json` lookup map. This completely eliminates complex backend hosting costs and guarantees O(1) instantaneous latency.

## ✨ Core Features

- **Content-Based ML Filtering:** A robust AI recommendation engine that predicts movie similarities using Natural Language Processing (NLP) techniques on shared metadata (genres, keywords, cast, and overview).
- **Offline ML Compute Pipeline:** The computational heavy lifting (CountVectorizer, Vector Math, & Cosine Similarity) is fully executed offline in Python to guarantee maximum frontend performance.
- **Ultra-Fast Delivery:** A Vanilla JS frontend communicates with the precomputed ML tensor mappings for `O(1)` zero-latency lookups.
- **Premium UI/UX:** Cinematic dark mode aesthetic with glassmorphism, smooth micro-animations, and dynamic visual states.
- **Serverless Architecture:** Deployable instantly and for free on Vercel or GitHub Pages without requiring Python server environments.

## 🧠 Machine Learning Architecture

CineMatch AI is a genuine **Machine Learning Recommendation Engine**. The architectural design simply shifts the execution phase from "on-demand" (compute at runtime) to "pre-computed" (compute at build time). Here is the detailed ML pipeline:

1. **Data Preprocessing & NLP:** The offline Python script (`backend/generate_similarity.py`) ingests a large movie dataset (`movie_list.pkl`), cleans the metadata, and concatenates relevant text features (genres, cast, keywords, overview) into a unified `tags` document for each movie.
2. **Vectorization (`CountVectorizer`):** Text documents are mathematically tokenized into dense numeric vectors. Scikit-Learn's `CountVectorizer` calculates the term frequency of every word across the entire corpus, generating a massive multi-dimensional mathematical matrix.
3. **Cosine Similarity Matrix Calculation:** To calculate how mathematically close two movies are, the engine calculates the **Cosine Similarity** (the angle between vectors) across the entire vector space. A smaller angle yields a similarity score closer to 1.0 (highly similar).
4. **Precomputed ML Model Export:** The matrix is evaluated offline, and the absolute closest nearest-neighbors (top recommendations) for every single movie are packaged into a lightweight `frontend/recommendations.json` tensor mapping.
5. **Instant Client Delivery:** In the browser, the JavaScript instantly queries this JSON mapping locally—delivering the results of complex matrix math with zero server compute costs.

## 🚀 Setup Instructions (Local)

### 1. Clone the Repository
```bash
git clone https://github.com/samarthtalwar9/Cinematch.git
cd Cinematch
```

### 2. Run the Application
Because the ML vector comparisons are already pre-computed in `recommendations.json`, you do **not** need to run a live Python Flask server!
Simply open `frontend/index.html` in your web browser or use VS Code Live Server.

## ☁️ Deployment Instructions

The project is built as a static artifact and can be deployed globally in 2 minutes:

### Vercel / GitHub Pages
1. Push your code to a GitHub repository.
2. Create a new project on [Vercel](https://vercel.com/) or enable [GitHub Pages](https://pages.github.com/).
3. Connect your repository.
4. Set the "Root Directory" to `frontend` (for Vercel).
5. Deploy! Your static site will instantly serve the HTML and ML JSON mappings globally.
